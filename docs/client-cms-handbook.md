# EnidPath Website Content Editing Guide

## Introduction

This guide is designed for non-technical users who need to update website content through the Payload CMS admin panel.

You do not need to edit code to make normal website updates.

Typical tasks include:

- changing homepage text
- updating contact details
- uploading images
- editing page content
- updating courses
- updating study programmes

## Where To Log In

Open the admin area in your browser:

```text
/admin
```

Sign in using your admin account.

## Main Content Areas

The website content is organised into these sections inside Payload:

- `Site Settings`
- `Home Page`
- `Pages`
- `Courses`
- `Programmes`
- `Media`

Each area controls a different part of the website.

## Recommended Editing Order

When making updates, use this order:

1. Upload images in `Media`
2. Update shared business details in `Site Settings`
3. Update homepage content in `Home Page`
4. Update standard pages in `Pages`
5. Update course entries in `Courses`
6. Update study programme entries in `Programmes`

This helps keep the website content organised and consistent.

## Site Settings

Use `Site Settings` to update content that appears across multiple parts of the website.

Examples:

- business name
- phone number
- email address
- WhatsApp number
- office address
- office hours
- navigation menu links
- footer links
- footer description
- footer disclaimer

If your organisation changes phone numbers, emails, navigation labels, or footer details, this is usually the first place to edit.

## Home Page

Use `Home Page` to manage the homepage content.

You can update:

- hero title
- hero subtitle
- hero image
- trust indicators
- statistics
- feature cards
- partnership section
- call-to-action text and buttons

If the homepage needs a refresh, start here.

## Pages

Use `Pages` to update the main website pages.

Each page is identified by a slug.

Current slugs include:

- `about`
- `services`
- `why-choose`
- `contact`
- `courses`
- `study`

Depending on the page, you may be able to edit:

- hero title
- hero subtitle
- section headings
- body paragraphs
- bullet lists
- feature cards
- call-to-action buttons

## Courses

Use `Courses` to manage the list shown on the Courses page.

You can:

- add a course
- edit a course
- remove a course

Typical fields:

- course name
- level
- credits

## Programmes

Use `Programmes` to manage the programme cards shown on the Study page.

Typical fields:

- programme level
- programme title
- description
- feature list

## Media

Use `Media` to upload and manage images.

Recommended approach:

1. upload the image first
2. add meaningful alt text
3. return to the page or setting where the image should be used
4. select the uploaded image

## Example Tasks

### Update the phone number

1. Open `Site Settings`
2. Update the phone number field
3. Save
4. Refresh the website and confirm the change appears in the header/footer/contact areas

### Change the homepage hero image

1. Open `Media`
2. Upload the new image
3. Open `Home Page`
4. Select the new hero image
5. Save
6. Refresh the homepage

### Update the About page text

1. Open `Pages`
2. Open the entry with slug `about`
3. Edit the text fields you need
4. Save
5. Refresh the About page on the website

### Add a new course

1. Open `Courses`
2. Create a new entry
3. Add the course name, level, and credits
4. Save
5. Refresh the Courses page

### Add or update a programme

1. Open `Programmes`
2. Create or edit the programme entry
3. Update title, description, and features
4. Save
5. Refresh the Study page

## Good Editing Habits

To keep the website consistent:

- keep headings short and clear
- keep subtitles easy to read
- avoid very long paragraphs
- check spelling before saving
- make sure links are correct
- upload images before selecting them
- avoid leaving list items blank

## After Every Update

After saving your changes:

1. open the related page on the website
2. refresh the page
3. confirm the new content appears correctly
4. test any updated links or buttons

## Troubleshooting

If your change does not appear:

- confirm you edited the correct item
- confirm the entry was saved
- refresh the website page
- confirm the correct slug or collection was used
- verify any images were uploaded properly

If something looks incomplete, a field may have been left blank or a list item may need to be filled in fully.

## Suggested Roles

Here is a simple way to think about who edits what:

- `Site Settings`: business owner or content manager
- `Home Page`: marketing or communications
- `Pages`: content editor
- `Courses`: academic or programme administrator
- `Programmes`: academic or programme administrator
- `Media`: anyone preparing approved website images

## Final Notes

The CMS is designed so that most normal website content can now be updated without development work.

If you are unsure where a piece of content belongs:

- shared business details usually belong in `Site Settings`
- homepage content belongs in `Home Page`
- standard page text belongs in `Pages`
- lists of courses belong in `Courses`
- study programme cards belong in `Programmes`
